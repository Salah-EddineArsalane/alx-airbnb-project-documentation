# Airbnb Clone – Use Case Diagram

This diagram shows how users interact with the system.

## Actors:
- **Guest**: Registers, logs in, books properties, makes payments, and leaves reviews.
- **Host**: Lists and manages properties, views bookings.
- **Admin**: Oversees users, properties, and bookings.

## Key Interactions:
- Guest ↔ System: Registration, booking, payment.
- Host ↔ System: Property creation and management.
- Admin ↔ System: Manage system operations.
